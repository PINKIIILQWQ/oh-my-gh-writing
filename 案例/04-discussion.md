# Discussion — 经典案例

---

## 精细版（完整 Discussion 模板）

### 案例 1：mozilla/rr (10k⭐)

rr 项目的 Discussion 模板聚焦技术设计讨论：

```markdown
### Problem
[当前系统的问题或限制]

### Proposed Solution
[提议的变更方案]

### Alternatives Considered
[其他被考虑的方案及不选理由]

### Open Questions
[尚未确定的点]
```

---

### 案例 2：jenkinsci/jenkins (23k⭐)

Jenkins 通过 JIRA + 邮件列表进行社区讨论：

```
标题前缀：[DISCUSS]
1. Background（背景）
2. Proposal（提案）
3. Open questions（征求意见点）
4. Vote（投票决定）
```

---

### 案例 3：chef/chef (8k⭐)

Chef RFC 讨论流程：

```markdown
## Motivation
## Specification
## Compatibility
## Rollback
## Reference Implementation
```

## 普通版（较短 Discussion 模板）

### 案例 4：hashicorp/vault (35k⭐)

Vault 的 Feature Description 也可用作 Discussion 框架：

```markdown
#### Feature Description
A written overview and why it solves challenges.

#### Use Case(s)
Describe the use case and deployment environments.
```
