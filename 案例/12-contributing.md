# CONTRIBUTING — 经典案例

---

## 精细版

### 案例 1：psf/black (39k⭐)
极简友好入口：将详细内容放在 ReadTheDocs，CONTRIBUTING.md本身只提供链接。

### 案例 2：redis/redis (68k⭐)
突出两大核心：
1. 贡献者许可协议（CLA）完整嵌入
2. Issue使用规范：仅用于Bug和功能请求，其他去Discord

### 案例 3：pytorch/pytorch (86k⭐)
最系统的贡献指南：Developing → Testing → Merging → Docs → CUDA → Windows → ASAN
包含AI-Assisted Development章节。

## 普通版

### 案例 4：apache/echarts (62k⭐)
多层面贡献指南（不仅代码）：Issue → PR → Debug → 邮件列表

### 案例 5：jupyter/notebook (13k⭐)
标准化格式：环境搭建 → 测试 → 代码风格（prettier+black）→ 文档

### 案例 6：python/mypy (19k⭐)
LLM辅助贡献政策（对LLM持中立态度，但新手不建议使用）+ 详细本地开发4步法。
