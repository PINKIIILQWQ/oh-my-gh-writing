- 新增 `pudding0503/github-badge-collection` 为第 6 个专项参考，并补入 README 徽章策略

## 优化空间

1. **补自动 runner**
   现在的全量测试依赖人工检查和静态样例。建议新增 `test/run-18-scenarios.md` 或脚本化 schema，明确每条测试提示对应的实际输出。

2. **对齐测试提示和对比样例**
   `test/README.md` 的 18 条提示和 `compare-18-scenarios.md` 的样例主题不完全一致。建议保留“新仓库双版对比”作为参考基线，再新增一份“按测试提示生成的实际回放”。

3. **给每个场景加机器可检查字段**
   可以在 reference 文件底部增加 `Required fields:` 列表，后续用脚本检查输出是否包含这些字段。

4. **README 场景继续增强**
   已加入 badge collection 思路。下一步可以补“项目类型到 badge 组合”的映射，例如 skill、CLI、库、GitHub Action、网页应用分别推荐哪些徽章。

5. **安装体验自动化**
   Codex 本地安装目前需要手动 `ln -s`。可以补一个 `scripts/install-codex-skill.sh`，负责检测已有安装、提示覆盖、创建符号链接。
