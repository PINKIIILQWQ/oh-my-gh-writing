# Emoji Guide

This is a quick local index of common GitHub artifact emoji. Use it when emoji is requested or the target repository already uses emoji. For complete coverage, consult gitmoji or the relevant project convention.

Source basis: gitmoji plus common GitHub artifact conventions.

## Commits and PR Titles

| Emoji | Code | Use | Meaning |
|-------|------|-----|---------|
| ✨ | `:sparkles:` | Feature | Introduce new features |
| 🐛 | `:bug:` | Bug fix | Fix a bug |
| 🚑️ | `:ambulance:` | Hotfix | Critical hotfix |
| ♻️ | `:recycle:` | Refactor | Refactor code |
| 🔥 | `:fire:` | Removal | Remove code or files |
| 🎨 | `:art:` | Structure / formatting | Improve code structure or format |
| ⚡️ | `:zap:` | Performance | Improve performance |
| 📝 | `:memo:` | Documentation | Add or update documentation |
| 💄 | `:lipstick:` | UI/style | Add or update UI and style files |
| ✅ | `:white_check_mark:` | Tests | Add, update, or pass tests |
| 🔧 | `:wrench:` | Config | Add or update configuration files |
| ⬆️ | `:arrow_up:` | Dependency | Upgrade dependencies |
| ⬇️ | `:arrow_down:` | Dependency | Downgrade dependencies |
| ➕ | `:heavy_plus_sign:` | Dependency | Add a dependency |
| ➖ | `:heavy_minus_sign:` | Dependency | Remove a dependency |
| 🚚 | `:truck:` | Move/rename | Move or rename resources |

| Emoji | Code | Use | Meaning |
|-------|------|-----|---------|
| 🚀 | `:rocket:` | Release | New release |
| 💥 | `:boom:` | Breaking change | Breaking behavior |
| 📦 | `:package:` | Package | Package release |
| 🔐 | `:closed_lock_with_key:` | Security | Security update |
| ⬆️ | `:arrow_up:` | Upgrade | Upgrade path |
| ⚠️ | `:warning:` | Warning | Migration risk |

## Rules

- Prefer semantic consistency over novelty.
- Do not mix emoji systems in one artifact.
- Do not use emoji when the target repository style is plain or formal.
- Do not infer emoji conventions when the repository has its own documented format.
