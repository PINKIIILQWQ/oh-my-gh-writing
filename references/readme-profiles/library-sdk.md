# README Profile: Library / SDK

## Use When

Use for reusable packages, SDKs, API clients, framework adapters, plugins, or libraries consumed from application code.

## Required Shape

- Overview: what developers can build with it.
- Installation: package manager commands and supported versions.
- Minimal usage: shortest working code example.
- Core concepts or API map: only the entry points most users need first.
- Compatibility: language/runtime/framework versions and supported environments.
- Links: full API docs, examples, changelog, contributing.

## Quick Start

Use install command plus a minimal copyable code snippet. If credentials or service setup are required, separate them from the minimal code path.

## Applicability

State language, runtime, framework, package manager, module format, browser/server support, auth model, and API version constraints.

## Badges/Visuals

Use registry version, CI, coverage, license, and downloads badges only with real sources. Avoid social badges unless community growth is part of the project.

## Avoid

- API reference dumps in the README.
- Unsupported framework compatibility claims.
- Examples that require hidden setup not described nearby.

## Evidence Required

Use package manifests, lockfiles, type definitions, examples, tests, official API docs, registry metadata, and user-provided compatibility notes.
