# README Profile: Design System / UI Kit

## Use When

Use for component libraries, design systems, UI kits, token packages, theme systems, or Figma/code component bridges.

## Required Shape

- Overview: supported product/design problem.
- Quick Start: install and render one component or import tokens.
- Components/tokens: concise inventory with links to full docs.
- Framework support: UI frameworks and styling stack.
- Theming/accessibility: supported modes and guarantees.
- Design assets: Figma/storybook/docs links only when real.

## Quick Start

Show one minimal component or token import. If peer dependencies are required, list them before the snippet.

## Applicability

State UI frameworks, styling system, component library, design tool support, theming modes, browser/runtime support, and accessibility claims.

## Badges/Visuals

Use package, CI, Storybook/docs, license, and design-tool badges when evidenced. Screenshots should show real components or tokens.

## Avoid

- Claiming accessibility compliance without tests or docs.
- Listing components that are not implemented.
- Treating design mockups as production support.

## Evidence Required

Use package manifests, component exports, token files, Storybook/docs, tests, design links, accessibility checks, and user-provided support matrix.
