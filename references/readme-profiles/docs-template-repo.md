# README Profile: Docs / Template Repository

## Use When

Use for documentation-only repositories, starter templates, boilerplates, rule collections, example repos, and reusable project skeletons.

## Required Shape

- Overview: what the docs/template helps users produce.
- Quick Start: copy, clone, generate, or browse path.
- What is included: concise file or module map.
- Applicability: intended audience, project types, required host format.
- Customization: what users should change first.
- Maintenance: source, license, contribution path.

## Quick Start

For templates, start with the safest copy/use path. For docs-only repos, start with navigation and entry pages instead of fake install commands.

## Applicability

State target artifact types, supported host format, intended audience, conversion/adaptation paths, and what the template does not cover.

## Badges/Visuals

Use license, status, docs, and validation badges when evidenced. Avoid package badges unless the template is published as a package.

## Avoid

- Presenting a template as an executable app.
- Overloading README with the full template contents.
- Linking files that do not exist.

## Evidence Required

Use repository file tree, template files, docs index, generator config, examples, license, and user-provided intended audience.
