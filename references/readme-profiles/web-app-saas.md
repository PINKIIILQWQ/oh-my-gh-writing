# README Profile: Web App / SaaS

## Use When

Use for hosted web apps, SaaS products, dashboards, admin tools, websites with application behavior, or self-hostable web services.

## Required Shape

- Overview: user problem and core workflows.
- Live/demo links: only if real.
- Quick Start: local development or self-host path.
- Configuration: required services, auth, env vars, databases.
- Deployment: supported platforms only when evidenced.
- Screenshots: use real assets when available.

## Quick Start

Start with local setup if the repository is developer-facing. For end-user SaaS READMEs, place live link and user docs before developer setup.

## Applicability

State user roles, browsers, deployment targets, backend/runtime, database, auth providers, API/data dependencies, and self-host limits.

## Badges/Visuals

Use deployment, CI, license, and status badges only with real services. Screenshots should show actual product state, not decorative mockups.

## Avoid

- Marketing hero copy as a substitute for usage.
- Claiming public SaaS availability without a real URL.
- Listing integrations not present in config or docs.

## Evidence Required

Use app config, package manifests, routes, deployment files, screenshots/assets, docs, official service docs, and user-provided URLs.
