# README Profile: Data / Model Project

## Use When

Use for datasets, ML models, benchmarks, notebooks, data pipelines, evaluation harnesses, or model cards hosted in a GitHub repository.

## Required Shape

- Overview: task, data/model purpose, and intended users.
- Access: download, checkpoint, dataset, or notebook path.
- Quick Start: minimal reproduce/infer/evaluate command.
- Data/model details: format, license, provenance, limitations.
- Metrics: only with source and evaluation setup.
- Hardware/runtime: required environment and constraints.

## Quick Start

Separate installation, data/model download, and first run. Mark heavy downloads, GPU requirements, or external credentials clearly.

## Applicability

State input/output formats, task type, model/runtime, hardware, dataset license, evaluation scope, privacy limitations, and intended/non-intended use.

## Badges/Visuals

Use dataset/model host badges, license, CI, paper, and benchmark badges only with real links. Do not invent performance metrics.

## Avoid

- Unverified accuracy or benchmark claims.
- Missing license/provenance for data.
- Hiding hardware or large download requirements.

## Evidence Required

Use model cards, dataset cards, notebooks, training/eval scripts, config, licenses, papers, benchmark logs, and user-provided results.
