# README — 精细标准

## 适用场景
项目首页文档，面向新用户快速上手。

## 精细标准（vscode, nextjs, tailwindcss）

### 结构
1. **标题**（项目名 + 一行描述）
2. **徽章栏**（构建状态 / 版本 / 许可 / 覆盖 / 下载量）
3. **简介**（2-3 句说明项目解决了什么问题）
4. **快速开始**（安装 + 最小示例，可复制粘贴）
5. **示例**（功能展示，含截图/GIF）
6. **文档导航**（链接到 Docs/API/教程）
7. **贡献指南**（链接到 CONTRIBUTING）
8. **许可**

### 参考仓库写法

**microsoft/vscode**
- 截图做首页 Hero，下载按钮突出
- 功能表格列出核心特性
- 文档家族导航完整（Docs/API/Extensions）
- 徽章栏 10+ 个

**vercel/nextjs**
- 快速开始极简：3 步构建一个页面
- 示例代码完全可复制
- 文档链接分层
- 徽章：npm / 构建 / 讨论

**tailwindlabs/tailwindcss**
- Live Play CDN 链接直接可试
- 功能列表 + 截图展示
- 干净无多余装饰

## README — 中等标准（deno, supabase, github/docs）

### 结构
标题→简介→快速开始→功能→文档链接→许可

### 参考仓库写法

**denoland/deno**
- 极简标题 + 安装命令
- 快速开始含完整示例
- 无徽章栏

**supabase/supabase**
- Hero 区域 + 快速开始
- 产品导向，文档链接分层
- 干净、现代

**github/docs**
- GitHub 风格：标题 + 描述 + 快速开始 + 配置
- 开发者友善

## 必含元素 Checklist
- [ ] 标题 + 一行描述
- [ ] 快速开始（安装 + 最小示例）
- [ ] 文档链接

## 可选项
- 徽章：默认 top 3（构建 + 版本 + 许可）
- TOC：默认自动生成
- 贡献者表：默认无
- GIF 演示：默认有（若产品有 UI）
