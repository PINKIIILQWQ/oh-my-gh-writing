# Emoji Guide — GitHub Artifact Quick Index

Use this file only when emoji are requested, the repository already uses emoji, or the target artifact clearly benefits from a community/product tone.

## 使用原则

- 默认不用 emoji。
- 只有用户要求、仓库已有风格如此，或 README/PR 明显适合社区、产品、教学语气时才少量使用。
- 一个标题最多 1 个 emoji。
- 不要用 emoji 替代准确标题。
- 严肃基础设施、企业工具、库文档默认不用 emoji。
- Commit message 只有目标项目采用 gitmoji 风格时才使用 emoji。

## Commits / PR Titles

| Emoji | Code | 场景 | 说明 |
|-------|------|------|------|
| ✨ | `:sparkles:` | 新功能 | Introduce new features |
| 🐛 | `:bug:` | 修 Bug | Fix a bug |
| 🚑️ | `:ambulance:` | 紧急热修 | Critical hotfix |
| ♻️ | `:recycle:` | 重构 | Refactor code |
| 🔥 | `:fire:` | 删除代码 | Remove code or files |
| 🎨 | `:art:` | 代码结构/格式 | Improve structure / format |
| ⚡️ | `:zap:` | 性能优化 | Improve performance |
| 📝 | `:memo:` | 文档 | Add or update documentation |
| 💄 | `:lipstick:` | UI/样式 | Add or update UI/style files |
| ✅ | `:white_check_mark:` | 测试 | Add, update, or pass tests |
| 🔧 | `:wrench:` | 配置文件 | Add or update configuration files |
| ⬆️ | `:arrow_up:` | 升级依赖 | Upgrade dependencies |
| ➕ | `:heavy_plus_sign:` | 加依赖 | Add a dependency |
| 🚚 | `:truck:` | 移动/重命名 | Move or rename resources |
| 💥 | `:boom:` | 破坏性变更 | Introduce breaking changes |
| 🔒️ | `:lock:` | 安全修复 | Fix security/privacy issues |
