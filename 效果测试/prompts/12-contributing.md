# 测试输入：CONTRIBUTING

## 用户请求

```
请为一个真实开源 React + TypeScript 组件库编写 CONTRIBUTING.md 草稿。

要求：
1. 先读取目标仓库的 package.json、测试命令、lint/typecheck 命令、CI 配置、现有 CONTRIBUTING 或 `.github/` 文件。
2. 再读取 1-2 个同类组件库的贡献指南作为结构参考。
3. 不要虚构 Node 版本、包管理器、coverage 目标、发布流程或 CI 要求。
4. 缺少证据的项目规范写成 `TODO`。
```

使用 oh-my-gh-writing 的 CONTRIBUTING 场景。
