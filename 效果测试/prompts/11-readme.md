```
1. 先读取目标仓库的 README、LICENSE、包配置文件、入口命令和 docs/examples（如存在）。

3. 输出时只使用目标仓库中能证实的功能、安装方式和命令。
4. 不要虚构 Homebrew/npm/cargo install、release 链接、平台支持或 benchmark。
5. 在最终回复中列出目标仓库和参考仓库链接，但不要写入 output.md。
```
